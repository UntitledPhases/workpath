# Raw vs Treated Rubric

Score each answer from 0 to 2 per dimension.

| Dimension | 0 | 1 | 2 |
| --- | --- | --- | --- |
| Decision framing | Does not restate the decision | Names the main recommendation | Names the decision, options, criteria, and missing context |
| Breadth before synthesis | Jumps to one answer | Compares options after deciding | Compares all options before final synthesis |
| Findings separation | Blends all reasoning together | Some separated reasoning | Distinct demand, economics, and risk findings |
| Structured comparison | No comparison structure | Partial bullet comparison | Clear compact table or equivalent |
| Risk handling | Generic or missing risks | Names plausible risks | Ties risks to option choice and execution |
| Actionability | Generic next steps | Some concrete steps | Time-bounded steps with operating metrics |
| Uncertainty discipline | No caveats | Some caveats | Explicit missing-context section |
| Workflow adherence | Not applicable or absent | Follows some workflow cues | Follows the intended protocol without reciting it |

The expected proof is not that the treated answer is always "right." The proof
is that the same natural-language task becomes more consistently structured when
a small Workpath workflow is injected.
